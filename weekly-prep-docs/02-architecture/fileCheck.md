# 前端体验优化需求（AI可执行版）

## 文档元信息
- 文档版本：V2
- 适用范围：仅前端（Vue 页面与样式层）
- 目标阶段：开发完成后的一轮统一优化
- 输入来源：当前项目实际使用体验反馈

## 全局约束
- 只处理前端页面展示与交互体验，不涉及后端接口改造。
- 优先保证移动端体验（手机、Pad），再考虑桌面端兼容。
- 所有优化需可复用、可维护，避免页面内重复样式定义。

## 优先级定义
- P0：必须优先完成，影响核心体验。
- P1：重要优化，可在 P0 后执行。
- P2：可延后优化。

## 问题清单（结构化）

### ITEM-001 页面整体视觉不一致
- 优先级：P0
- 当前问题：
  - 当前页面主要依赖默认 Vant 样式，整体视觉与目标原型差异较大。
- 目标结果：
  - 页面样式与移动端原型保持高度一致（布局、间距、颜色、字体、圆角、阴影、组件状态）。
- 参考基线：
  - `./01-product/product-prototype-mobile.html`
- 验收标准：
  - 关键页面与原型视觉差异明显减少（人工对照检查通过）。
  - 同类组件在不同页面表现一致（按钮、卡片、表单、导航等）。
- 建议动作：
  - 提炼设计 Token（颜色、字号、间距、圆角、阴影、层级）。
  - 对现有组件进行主题化覆盖，而非页面内零散覆盖。

### ITEM-002 样式组织分散，缺乏统一规范
- 优先级：P0
- 当前问题：
  - 样式大量写在页面局部，缺乏全局风格层与可复用规则。
- 目标结果：
  - 建立统一样式目录与分层机制，页面仅保留本页特有样式。
- 建议目录（示例）：
  - `src/styles/tokens.scss`（设计变量）
  - `src/styles/mixins.scss`（复用 mixin）
  - `src/styles/base.scss`（重置与基础元素）
  - `src/styles/components/*.scss`（通用组件样式）
- 验收标准：
  - 新增或修改页面时，优先复用全局样式资产。
  - 页面内样式体量明显下降，重复样式减少。
  - 样式职责清晰：全局通用 vs 页面特有可区分。

### ITEM-003 移动端多设备适配不足
- 优先级：P0
- 当前问题：
  - 项目主要在手机与 Pad 使用，不同分辨率下展示效果不稳定。
- 目标结果：
  - 在常见移动分辨率下保持一致的可读性与可操作性。
- 建议动作：
  - 采用统一适配方案（如 viewport + 相对单位 + 断点策略）。
  - 定义关键断点（手机竖屏、手机横屏、Pad）并形成通用响应式规则。
  - 对关键页面进行真实设备或模拟器回归验证。
- 验收标准：
  - 核心流程页面在手机与 Pad 均无明显布局错位、文字溢出、点击区域过小问题。
  - 横竖屏切换后页面结构可用，不出现核心操作丢失。

## AI执行指令（给后续开发/生成工具）
- 按优先级从 P0 到 P2 执行。
- 每个 ITEM 输出：
  - 改造方案（涉及文件、影响范围）
  - 实际改动清单（按文件列出）
  - 验收结果（逐条对应验收标准）
  - 风险与回滚点
- 若与现有实现冲突，优先满足「视觉一致性 + 可维护性 + 移动端可用性」。
