# 周备有谱 PRD（AI执行版）

## 文档元信息
- 文档版本：V1.0
- 业务阶段：个人自用验证（MVP）
- 执行周期：2 周开发 + 4 周验证
- 对齐文档：`README.md`、`plan.md`、`bugFix.md`、`fileCheck.md`、`开发计划与验收SOP.md`

## 1. 产品目标（唯一主目标）
- 在每周固定时段，10 分钟内完成一周菜单与购物决策，降低漏买与浪费。

## 2. 用户与场景
- 核心用户：单用户（本人）
- 次级用户：3-5 位同类生活场景用户（上班族/小家庭）
- 使用场景：周末统一计划，工作日快速执行

## 3. 问题定义
- `PAIN-001`：一周吃什么决策耗时高
- `PAIN-002`：购物易漏买、重复买
- `PAIN-003`：库存不可见，食材浪费
- `PAIN-004`：工作日补货频繁，节奏被打断

## 4. 功能范围（MVP）

### 4.1 Must（必须做）
- `FEAT-001` 周计划创建：人数、天数、预算、偏好
- `FEAT-002` 菜单管理：模板库 + 手动替换（首版 30~50 道）
- `FEAT-003` 购物清单自动生成：别名归一、单位换算、同类聚合、分组展示
- `FEAT-004` 清单手动新增：同名同单位默认合并
- `FEAT-005` 库存扣减：勾选“家里已有”触发库存变更
- `FEAT-006` 采购执行：三态 `pending/purchased/from_inventory` 互斥切换
- `FEAT-007` 下周复用：白名单复制 + 执行态重置
- `FEAT-008` 数据备份恢复：JSON 导出/导入（全量覆盖 + 导入前快照）

### 4.2 Should（应做）
- `FEAT-009` 周提醒
- `FEAT-010` 备菜引导
- `FEAT-011` 耗时标签（15/30/45 分钟）
- `FEAT-012` 预算提示（`actualCost > budget` 高亮）

### 4.3 Won't（本期不做）
- 配送与电商下单
- 社区评论与分享
- 复杂营养系统
- 会员与付费体系
- 自动价格抓取与预测

## 5. 主流程（唯一主链路）
- `FLOW-001`
  - 首页 -> 新建计划 -> 生成菜单 -> 调整菜单 -> 生成清单 -> 库存扣减 -> 采购执行 -> 周复盘 -> 复制下周

## 6. 页面定义（最小页面集）
- `PAGE-001` 首页：本周状态、创建计划、复制上周
- `PAGE-002` 计划页：人数/天数/预算/偏好输入
- `PAGE-003` 菜单页：周视图卡片、替换/删除、流程步骤展示
- `PAGE-004` 清单页：分组清单、状态筛选、手动新增、库存扣减
- `PAGE-005` 复盘页：耗时、漏买、浪费、预算对比

## 7. 关键业务规则（与架构文档强一致）
- `RULE-001` 聚合顺序固定：别名归一 -> 单位换算 -> 个转基准单位 -> 聚合
- `RULE-002` 基准单位固定：固体 `g`、液体 `ml`
- `RULE-003` 库存变更必须走 `InventoryLog`
- `RULE-004` 同一来源通过 `sourceId` 幂等，禁止重复扣减
- `RULE-005` 复制周计划只复制白名单字段，执行态全部重置
- `RULE-006` 导入仅支持全量覆盖；导入前必须自动快照

## 8. 数据对象（业务口径）
- `Plan`
- `MealItem`
- `ShoppingItem`
- `InventoryItem`
- `InventoryLog`
- `WeeklyReview`
- `ingredientAliasMap` / `unitConvertMap` / `pieceToGramMap`

> 详细字段定义以 `tech-architecture.md` 为准。

## 9. 验收标准（开发期）
- `AC-001` 可创建并保存完整周计划
- `AC-002` 菜单至少可替换 1 次并持久化
- `AC-003` 清单可完成归一、换算、聚合
- `AC-004` 手动新增食材可正确归类并参与统计
- `AC-005` 库存扣减写日志，反勾选可回滚
- `AC-006` 三态切换互斥并可回退
- `AC-007` 复制下周后执行态重置
- `AC-008` JSON 导出/导入可恢复核心数据
- `AC-009` 导入前自动快照，导入后展示恢复成功与时间

## 10. 验证指标（4 周）
- `KPI-001` `planningMinutes`（目标下降 >= 40%）
- `KPI-002` `shoppingMinutes`
- `KPI-003` `missedItemsCount`（目标显著下降）
- `KPI-004` `wastedItemsCount`（目标下降 >= 30%）
- `KPI-005` `completionRate`
- `KPI-006` `nextWeekRetention`

## 11. Go / No-Go 判定

### Go
- 连续 4 周真实使用
- `planningMinutes` 下降 >= 40%
- 漏买次数显著下降
- 浪费食材下降 >= 30%

### No-Go
- 无法连续 4 周使用
- 核心指标无明显改善
- 高频问题在 1 个迭代内无法收敛

## 12. AI执行指令
- 按 `FEAT-001 -> FEAT-008` 顺序优先实现 Must。
- 每次提交必须输出：
  - 对应 FEAT-ID / RULE-ID / AC-ID
  - 影响文件列表
  - 测试结果（单测/集成/E2E）
  - 风险与回滚方案
