# 周备有谱方案文档（AI执行版）

## 文档元信息
- 文档版本：V2
- 对齐基线：`demo.md v1.0`、`bugFix.md v2`、`tech-architecture.md v2`
- 发布口径：P0 清零 + 主链路 E2E 通过 + 里程碑全部通过
- 统计周期：按周

## 1. 可行性结论（结论先行）
- 方向可行：每周买菜/做饭是高频刚需。
- 差异化清晰：聚焦“每周备菜决策 + 清单执行闭环”，不做泛电商。
- MVP 可落地：先做计划、菜单、清单、库存、复盘闭环。
- 核心风险可控：通过模板库、周复制、规则引擎降低冷启动与流失。

## 2. 产品定位（结构化）

### 2.1 目标用户
- 25-40 岁上班族或小家庭
- 每周做饭 >= 3 次
- 关注省时间、省钱、减少浪费

### 2.2 核心痛点
- 一周吃什么决策成本高
- 买菜易买漏、买多、买错
- 工作日时间紧，频繁临时补货

### 2.3 一句话价值
- 10 分钟完成一周菜单和清单，工作日 15-30 分钟完成做饭。

## 3. PRD 范围定义

### 3.1 V1 产品目标
- 建立“周计划 -> 菜单 -> 清单 -> 库存 -> 执行 -> 复盘”闭环。

### 3.2 V1 必做能力（Must）
- 周计划创建（人数、做饭日期、预算、偏好）
- 周计划生命周期：单周单计划（`weekStartDate` 唯一），已有计划进入编辑模式
- 周计划删除：支持删除本周计划（级联删除菜单/清单/周复盘数据，需二次确认）
- 菜单生成来源：模板库 + 手动替换
- 模板库首版：30~50 道高频家常菜
- 清单自动生成：别名归一 + 单位换算 + 聚合 + 分组
- 清单支持手动新增食材（同名同单位自动合并）
- 购物项三态：`pending` / `purchased` / `from_inventory`
- 库存扣减与回滚：`InventoryLog` + `sourceId` 幂等
- 下周复制：白名单复制 + 执行态重置
- 数据备份恢复：JSON 导出/导入（全量覆盖）
- 导入恢复安全：覆盖前自动快照 + 二次确认 + 成功提示

### 3.3 V1 应做能力（Should）
- 备菜步骤引导
- 菜谱耗时标签（15/30/45 分钟）
- 预算对比提示（`budget` vs `actualCost`）
- 每周提醒

### 3.4 V1 不做（Won't）
- 自营配送履约
- 社区/短视频
- 复杂营养系统
- 多人协作聊天
- 自动价格抓取与复杂预算预测

## 4. 主流程定义（AI流程节点）
- `home` -> `plan_create_or_edit` -> `menu_generate` -> `list_generate` -> `inventory_deduct` -> `shopping_execute` -> `prep_execute` -> `weekly_review`

- 菜单页：每日菜品卡片、替换、生成清单

## 7. 风险与应对（结构化）

### RISK-001 留存风险
- 风险：首周后流失
- 应对：模板复用 + 提醒 + 周复盘反馈（省时/省钱）

### RISK-002 内容质量风险
- 风险：菜谱与食材数量不准
- 应对：先上 30~50 高频模板，保留手动替换

### RISK-003 数据一致性风险
- 风险：库存重复扣减、取消不回滚
- 应对：`InventoryLog` 流水 + `sourceId` 幂等 + 回滚规则

### RISK-004 清单聚合准确性风险
- 风险：别名和单位不一致导致误合并
- 应对：`ingredientAliasMap` + `unitConvertMap` + `pieceToGramMap`

### RISK-005 范围失控风险
- 风险：功能膨胀导致延期
- 应对：严格执行 Must/Should/Won't 边界

### RISK-006 数据丢失风险
- 风险：本地使用中误删或迁移丢失
- 应对：IndexedDB 主存储 + 每周导出 + 导入前快照

# 周备有谱（每周快速备菜）方案文档

## 一、可行性结论

- 方向可行，且需求高频：每周都要买菜、做饭，具备稳定使用场景。
- 差异化空间明确：不做“泛买菜电商”，聚焦“每周备菜决策 + 清单执行”。
- MVP 可快速落地：先做计划、清单、库存、执行闭环，不依赖自建履约。
- 核心风险可控：主要在内容冷启动与留存，通过模板化和周复用机制缓解。

---

## 二、产品定位

### 1）目标用户
- 25-40 岁上班族 / 小家庭
- 每周做饭 3 次以上
- 关注省时间、省钱、减少浪费

### 2）核心痛点
- 不知道一周吃什么，决策成本高
- 买菜容易买漏、买多、买错
- 工作日没时间，临时补货频繁

### 3）一句话价值
**10 分钟完成一周菜单和购物清单，平日 15-30 分钟快速做饭。**

---

## 三、PRD（V1，对齐 demo v0.2 与 bugFix v1）

### 1）产品目标
围绕“每周备菜”建立闭环：
1. 周计划生成  
2. 自动清单整理  
3. 库存扣减 + 执行追踪  

### 2）核心功能（V1）
- 周菜单规划（人数、做饭日期、口味、预算）
- 周计划生命周期：单周单计划（同一周仅保留 1 条计划），已有计划默认进入编辑
- 周计划删除：支持删除当前周计划，删除时同步清理关联菜单/清单/周复盘（需二次确认）
- 菜单生成来源（模板库 + 手动替换，不做智能推荐）
- 首版提供 30~50 道高频家常菜模板，覆盖常见午/晚餐场景
- 自动购物清单（别名归一 + 单位换算 + 同类合并 + 按品类排序）
- `pieceToGramMap` 首版覆盖 30~50 个高频食材，超出范围提示手动确认单位
- 库存管理（已有食材扣减、可回滚、幂等）
- 备菜指引（周末预处理 + 工作日快做）
- 模板复用（下周一键复制，白名单复制 + 状态重置）
- 数据备份（JSON 导出与导入恢复，仅支持全量覆盖）
- 导入恢复流程：覆盖前自动导出当前快照，覆盖后提示恢复成功与恢复时间

### 3）V1 不做
- 自营配送
- 社区/短视频
- 复杂营养体系
- 多人协作与聊天
- 自动价格抓取与复杂预算预测

### 4）主流程
首页 -> 新建/编辑计划 -> 生成菜单 -> 生成清单 -> 库存扣减 -> 采购 -> 备菜执行 -> 周复盘

### 5）成功指标（V1）
- 计划创建转化率 >= 60%
- 清单生成率 >= 80%
- 周计划完成率 >= 35%
- 次周留存 >= 25%
- 数据恢复成功率 = 100%（导入后核心数据可用）

## 五、MVP 优先级（MoSCoW）

### Must（必须）
- 周计划创建
- 菜单生成与替换
- 清单自动合并（含别名/单位规则）
- 清单支持手动新增食材（同名同单位自动合并）
- 库存扣减（含流水与回滚）
- 采购三态状态（待购买/已购买/冰箱已有，互斥）
- 周计划保存/复用（白名单复制）
- JSON 导出/导入

### Should（应该）
- 菜谱耗时标签（15/30/45 分钟）
- 备菜步骤引导
- 预算对比提示（budget vs actualCost）
- 周提醒

### 数据口径（V1）
- `actualCost` 记入周复盘数据，与 `budget` 做对比并给出超预算高亮提示。

### Could（可选）
- 家庭共享
- 营养标签
- 语音录入库存
- 清单导出图片
- 自定义口味偏好录入与管理（后期优化）

### Won’t（本期不做）

## 八、风险与应对

### 1）留存风险
- 风险：首周使用后流失  
- 应对：模板复用、自动提醒、周复盘反馈（省时/省钱）

### 2）内容质量风险
- 风险：菜谱与食材数量不准  
- 应对：先上线 30~50 道高频家常菜模板，并通过手动替换保证可用性

### 2.1）数据一致性风险
- 风险：库存重复扣减、反勾选不回滚  
- 应对：新增 `InventoryLog`，所有库存变更走流水，按 `sourceId` 幂等

### 2.2）清单聚合准确性风险
- 风险：别名、单位不统一导致合并错误  
- 应对：上线 `ingredientAliasMap`、`unitConvertMap`、`pieceToGramMap`

### 3）履约依赖风险
- 风险：接入配送体系复杂  
- 应对：先做清单工具，后期对接第三方跳转下单

### 4）范围失控风险
- 风险：功能过多导致延期  
- 应对：严格控制 MVP 边界，不做社区和重运营模块

### 5）数据丢失风险
- 风险：单用户本地使用中数据意外丢失  
- 应对：IndexedDB 作为主存储 + 每周 JSON 导出备份 + 导入前自动快照

## 十二、关键节点验证与每周验收 Checklist

### 1）关键节点验证办法（里程碑）

| 里程碑 | 验证动作 | 通过标准 | 失败处理 |
|---|---|---|---|
| M1 规则引擎正确性（W1D5） | 用别名+多单位+“个”混合样例生成清单 | 执行顺序固定（别名->换算->个转 g->聚合），结果一致 | 标记 `needsManualConfirm`，禁止静默合并 |
| M2 库存幂等回滚（W2D2） | 同一清单项连续勾选/取消 5 次 | 无重复扣减；取消后库存可回滚；日志前后值一致 | 锁定该项并提示异常，进入缺陷修复 |
| M3 周复制规则（W2D3） | 复制上周到新周并逐字段比对 | 白名单字段复制；执行态全部重置 | 复制失败不落库，保持旧数据不变 |
| M4 导入恢复可靠性（W2D4） | 导出->清空->导入->全量比对 | 导入前快照存在；导入后核心数据可用 | 自动提示回退到快照 |
| M5 主链路验收（W2D6） | 跑完整链路 E2E | 创建->菜单->清单->库存->采购->复制 全通过 | 阻断发布，先清零 P0 |
